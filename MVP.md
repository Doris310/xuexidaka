# 纯前端离线学习任务打卡网页应用 MVP 需求文档

## 1. 项目概述

### 1.1 产品定位

本项目是一个纯前端、离线可用的学习任务打卡网页应用，用于帮助用户按学科管理学习任务、完成每日打卡、使用计时器进行专注学习，并查看周总结与月总结。

应用不依赖后端服务，不需要账号系统，不需要联网即可使用。所有数据保存在浏览器本地。

### 1.2 MVP 目标

MVP 阶段需要实现一个可独立使用的学习打卡工具，重点满足以下需求：

- 管理多个学科。
- 为不同学科创建学习任务。
- 对任务进行每日打卡。
- 使用计时器记录学习时长，并在时间到后响铃提醒。
- 查看本周学习完成情况。
- 查看本月学习完成情况。
- 数据离线保存，刷新页面后不丢失。

### 1.3 非目标范围

MVP 阶段暂不实现以下功能：

- 用户注册、登录、云同步。
- 多设备同步。
- 后端数据库。
- 复杂权限管理。
- 课程表导入。
- AI 学习建议。
- 任务协作或分享。
- 浏览器通知权限管理之外的复杂提醒系统。
- PWA 安装能力。后续可扩展，但 MVP 不强制要求。

## 2. 技术选型

### 2.1 基础技术

项目采用纯前端技术实现：

- HTML：负责页面结构。
- CSS：负责页面样式、布局、响应式适配。
- JavaScript：负责交互逻辑、数据处理、计时器、LocalStorage 读写。
- LocalStorage：负责本地持久化存储。

### 2.2 技术约束

- 不使用后端服务。
- 不使用数据库服务。
- 不依赖构建工具。
- 不依赖前端框架。
- 不依赖网络资源。
- 所有 CSS、JS、音频提醒资源应尽量本地化。
- 页面应可以通过直接打开 HTML 文件运行。

### 2.3 文件规划建议

MVP 推荐使用以下文件结构：

```text
/
  index.html
  styles.css
  app.js
  assets/
    alarm.mp3
```

说明：

- `index.html`：页面结构入口。
- `styles.css`：全局样式与组件样式。
- `app.js`：应用状态、数据操作、渲染与事件处理。
- `assets/alarm.mp3`：计时结束响铃音频。

如果希望进一步简化，也可以在 MVP 第一版中只使用单个 `index.html`，但后续维护性较差，不推荐作为最终开发依据。

## 3. 用户画像与使用场景

### 3.1 目标用户

- 学生。
- 自学者。
- 备考用户。
- 需要按学科规划学习任务的人。

### 3.2 核心使用场景

1. 用户添加多个学科，例如语文、数学、英语、物理。
2. 用户为某个学科添加学习任务，例如“完成数学错题整理 30 分钟”。
3. 用户当天完成任务后点击打卡。
4. 用户使用计时器进行专注学习，时间结束后响铃提醒。
5. 用户在周末查看本周各学科完成情况。
6. 用户在月底查看本月学习总览。

## 4. 页面结构规划

### 4.1 整体页面布局

应用采用单页结构，包含以下主要区域：

- 顶部栏。
- 今日概览区。
- 学科管理区。
- 任务打卡区。
- 计时器区。
- 周总结区。
- 月总结区。

页面不需要路由系统。各功能区可以在同一页面内纵向排列，也可以使用顶部标签切换展示。MVP 推荐使用标签切换，避免页面过长。

### 4.2 顶部栏

顶部栏内容：

- 应用名称：学习打卡。
- 当前日期。
- 今日完成任务数。
- 今日学习时长。

顶部栏行为：

- 日期根据用户本地时间自动显示。
- 今日完成任务数从当天打卡记录统计。
- 今日学习时长从当天计时记录统计。

### 4.3 页面导航

推荐使用标签导航：

- 今日任务。
- 学科管理。
- 计时器。
- 周总结。
- 月总结。

交互要求：

- 点击标签切换当前显示区域。
- 当前标签需要有明显选中状态。
- 切换标签不刷新页面。

### 4.4 今日任务页面

今日任务页面用于展示和完成学习任务。

内容包括：

- 今日概览卡片。
- 任务筛选控件。
- 任务列表。
- 新增任务入口。

任务筛选条件：

- 全部任务。
- 未完成。
- 已完成。
- 按学科筛选。

任务卡片展示内容：

- 任务名称。
- 所属学科。
- 预计学习时长。
- 任务状态。
- 今日是否已打卡。
- 创建日期。

任务卡片操作：

- 打卡完成。
- 取消今日打卡。
- 编辑任务。
- 删除任务。

### 4.5 学科管理页面

学科管理页面用于维护学科列表。

内容包括：

- 学科列表。
- 新增学科表单。
- 学科统计摘要。

学科卡片展示内容：

- 学科名称。
- 学科颜色。
- 学科备注。
- 当前任务数量。
- 本周完成次数。
- 本月完成次数。

学科操作：

- 新增学科。
- 编辑学科。
- 删除学科。

删除学科规则：

- 如果该学科下没有任务，可以直接删除。
- 如果该学科下存在任务，需要二次确认。
- 删除学科时，其关联任务的处理方式需要固定为：同时删除该学科下所有任务及相关打卡记录。

### 4.6 计时器页面

计时器页面用于进行专注学习计时。

内容包括：

- 学科选择。
- 任务选择。
- 计时时长输入或快捷选项。
- 倒计时显示。
- 开始、暂停、继续、重置按钮。
- 计时结束响铃。
- 本次计时完成后的记录确认。

计时时长快捷选项：

- 15 分钟。
- 25 分钟。
- 30 分钟。
- 45 分钟。
- 60 分钟。

计时器状态：

- 未开始。
- 运行中。
- 已暂停。
- 已完成。

计时结束后行为：

- 播放响铃音频。
- 显示完成提示。
- 记录实际学习时长。
- 如果已选择任务，允许用户选择是否同时将该任务标记为今日完成。

### 4.7 周总结页面

周总结页面用于展示当前自然周的学习情况。

统计范围：

- 以本地时间为准。
- 周一作为一周开始。
- 周日作为一周结束。

内容包括：

- 本周完成任务总数。
- 本周学习总时长。
- 各学科完成次数。
- 各学科学习时长。
- 每日完成任务数量。
- 每日学习时长。

展示形式：

- 统计卡片。
- 简单列表。
- 简易条形图。

MVP 不要求使用第三方图表库，条形图可以使用 CSS 宽度实现。

### 4.8 月总结页面

月总结页面用于展示当前自然月的学习情况。

统计范围：

- 以本地时间为准。
- 每月 1 日作为开始。
- 当月最后一天作为结束。

内容包括：

- 本月完成任务总数。
- 本月学习总时长。
- 本月打卡天数。
- 各学科完成次数。
- 各学科学习时长。
- 连续打卡天数。
- 月度每日打卡概览。

展示形式：

- 统计卡片。
- 学科排行列表。
- 月历式打卡视图。

## 5. 数据结构定义

### 5.1 LocalStorage Key

LocalStorage 使用一个主 Key 存储完整应用数据：

```text
study_checkin_app_data
```

推荐使用单一 JSON 对象保存全部数据，便于统一读取、迁移和备份。

### 5.2 根数据结构

```json
{
  "version": 1,
  "subjects": [],
  "tasks": [],
  "checkins": [],
  "timerLogs": [],
  "settings": {}
}
```

字段说明：

- `version`：数据版本号，用于后续数据迁移。
- `subjects`：学科列表。
- `tasks`：任务列表。
- `checkins`：任务打卡记录。
- `timerLogs`：计时记录。
- `settings`：应用设置。

### 5.3 学科数据结构

```json
{
  "id": "subject_唯一ID",
  "name": "数学",
  "color": "#4F7CFF",
  "note": "重点复习错题和函数",
  "createdAt": "2026-06-09T10:00:00.000Z",
  "updatedAt": "2026-06-09T10:00:00.000Z"
}
```

字段说明：

- `id`：学科唯一标识。
- `name`：学科名称，必填。
- `color`：学科主题色。
- `note`：备注，可为空。
- `createdAt`：创建时间，ISO 字符串。
- `updatedAt`：更新时间，ISO 字符串。

校验规则：

- 学科名称不能为空。
- 学科名称建议不超过 20 个字符。
- 同名学科不允许重复创建。
- 学科颜色必须有默认值。

### 5.4 任务数据结构

```json
{
  "id": "task_唯一ID",
  "subjectId": "subject_唯一ID",
  "title": "完成函数错题整理",
  "description": "整理最近三套卷子的函数错题",
  "estimatedMinutes": 30,
  "isActive": true,
  "createdAt": "2026-06-09T10:00:00.000Z",
  "updatedAt": "2026-06-09T10:00:00.000Z"
}
```

字段说明：

- `id`：任务唯一标识。
- `subjectId`：所属学科 ID。
- `title`：任务名称，必填。
- `description`：任务说明，可为空。
- `estimatedMinutes`：预计学习时长，单位分钟。
- `isActive`：是否启用。MVP 中删除任务可以直接物理删除，也可以将其置为 false。推荐 MVP 使用物理删除，降低复杂度。
- `createdAt`：创建时间，ISO 字符串。
- `updatedAt`：更新时间，ISO 字符串。

校验规则：

- 任务名称不能为空。
- 任务名称建议不超过 40 个字符。
- 必须选择一个学科。
- 预计学习时长必须为正整数。

### 5.5 打卡记录数据结构

```json
{
  "id": "checkin_唯一ID",
  "taskId": "task_唯一ID",
  "subjectId": "subject_唯一ID",
  "date": "2026-06-09",
  "createdAt": "2026-06-09T10:30:00.000Z"
}
```

字段说明：

- `id`：打卡记录唯一标识。
- `taskId`：关联任务 ID。
- `subjectId`：关联学科 ID，冗余保存，便于统计。
- `date`：打卡日期，格式为 `YYYY-MM-DD`，以本地日期为准。
- `createdAt`：打卡创建时间，ISO 字符串。

业务规则：

- 同一个任务在同一天只能有一条打卡记录。
- 取消今日打卡时，删除对应日期和任务的打卡记录。
- 删除任务时，需要删除该任务对应的打卡记录。
- 删除学科时，需要删除该学科对应的任务和打卡记录。

### 5.6 计时记录数据结构

```json
{
  "id": "timer_唯一ID",
  "subjectId": "subject_唯一ID",
  "taskId": "task_唯一ID或null",
  "plannedMinutes": 25,
  "actualMinutes": 25,
  "startedAt": "2026-06-09T10:00:00.000Z",
  "endedAt": "2026-06-09T10:25:00.000Z",
  "date": "2026-06-09"
}
```

字段说明：

- `id`：计时记录唯一标识。
- `subjectId`：关联学科 ID。
- `taskId`：关联任务 ID，可为空。
- `plannedMinutes`：计划计时时长。
- `actualMinutes`：实际完成时长。
- `startedAt`：开始时间。
- `endedAt`：结束时间。
- `date`：记录日期，格式为 `YYYY-MM-DD`，以本地日期为准。

业务规则：

- 只有计时完成后才写入计时记录。
- 暂停期间不计入剩余时间消耗。
- 重置未完成的计时器时，不写入计时记录。
- 用户手动停止未完成计时器时，MVP 默认不记录。

### 5.7 设置数据结构

```json
{
  "alarmEnabled": true,
  "defaultTimerMinutes": 25,
  "weekStartsOn": 1
}
```

字段说明：

- `alarmEnabled`：是否启用计时结束响铃。
- `defaultTimerMinutes`：默认计时时长。
- `weekStartsOn`：一周起始日。MVP 固定为 1，表示周一。

## 6. 核心功能逻辑描述

### 6.1 应用初始化

初始化流程：

1. 页面加载。
2. 从 LocalStorage 读取 `study_checkin_app_data`。
3. 如果数据不存在，创建默认数据结构。
4. 如果数据存在，解析 JSON。
5. 如果解析失败，提示用户本地数据异常。
6. 渲染页面。

默认数据建议：

- 学科列表为空。
- 任务列表为空。
- 打卡记录为空。
- 计时记录为空。
- 默认计时时长为 25 分钟。
- 默认响铃开启。

### 6.2 数据保存

所有新增、编辑、删除、打卡、取消打卡、计时完成操作后，都需要立即保存到 LocalStorage。

保存流程：

1. 修改内存中的应用状态。
2. 将完整状态序列化为 JSON。
3. 写入 LocalStorage。
4. 重新渲染受影响区域。

### 6.3 学科管理逻辑

新增学科：

1. 用户输入学科名称。
2. 用户选择或使用默认颜色。
3. 校验名称是否为空。
4. 校验名称是否重复。
5. 创建学科对象。
6. 保存数据。
7. 更新学科列表和相关筛选项。

编辑学科：

1. 用户选择学科。
2. 修改名称、颜色或备注。
3. 执行校验。
4. 更新 `updatedAt`。
5. 保存数据。
6. 重新渲染。

删除学科：

1. 用户点击删除。
2. 判断该学科下是否存在任务。
3. 如果存在任务，展示二次确认。
4. 用户确认后，删除该学科、关联任务、关联打卡记录、关联计时记录。
5. 保存数据。
6. 重新渲染。

### 6.4 任务管理逻辑

新增任务：

1. 用户选择学科。
2. 用户输入任务名称。
3. 用户输入预计学习时长。
4. 用户可填写任务说明。
5. 校验必填项。
6. 创建任务对象。
7. 保存数据。
8. 更新任务列表。

编辑任务：

1. 用户选择任务。
2. 修改任务名称、学科、预计时长或说明。
3. 校验数据。
4. 更新 `updatedAt`。
5. 保存数据。
6. 重新渲染。

删除任务：

1. 用户点击删除。
2. 展示二次确认。
3. 用户确认后，删除任务、关联打卡记录、关联计时记录。
4. 保存数据。
5. 重新渲染。

### 6.5 打卡逻辑

今日打卡：

1. 用户在任务卡片点击“打卡”。
2. 获取当前本地日期，格式为 `YYYY-MM-DD`。
3. 判断该任务当天是否已有打卡记录。
4. 如果没有，则创建打卡记录。
5. 保存数据。
6. 更新任务状态、今日概览、周总结、月总结。

取消今日打卡：

1. 用户点击“取消打卡”。
2. 查找当前日期下该任务对应的打卡记录。
3. 删除该打卡记录。
4. 保存数据。
5. 重新渲染。

已完成状态判断：

- 如果 `checkins` 中存在 `taskId` 等于当前任务 ID 且 `date` 等于今天的记录，则该任务今日已完成。
- 否则为今日未完成。

### 6.6 计时器逻辑

开始计时：

1. 用户选择学科。
2. 用户可选择任务。
3. 用户选择或输入计时时长。
4. 点击开始。
5. 计时器进入运行中状态。
6. 页面每秒更新剩余时间。

暂停计时：

1. 用户点击暂停。
2. 停止倒计时更新。
3. 保留当前剩余时间。
4. 状态变为已暂停。

继续计时：

1. 用户点击继续。
2. 从当前剩余时间继续倒计时。
3. 状态变为运行中。

重置计时：

1. 用户点击重置。
2. 停止计时。
3. 剩余时间恢复为设定时长。
4. 状态变为未开始。
5. 不写入计时记录。

计时完成：

1. 剩余时间到 0。
2. 停止计时。
3. 播放响铃。
4. 创建计时记录。
5. 如果选择了任务，提示是否将任务标记为今日完成。
6. 保存数据。
7. 更新今日概览、周总结、月总结。

响铃规则：

- 如果 `settings.alarmEnabled` 为 true，则播放音频。
- 音频播放失败时，应显示视觉提醒。
- 响铃后需要提供停止提示或自动结束音频。

### 6.7 周总结逻辑

周总结统计当前自然周数据。

计算步骤：

1. 获取当前本地日期。
2. 计算本周周一和周日日期。
3. 过滤 `checkins` 中日期位于本周范围内的数据。
4. 过滤 `timerLogs` 中日期位于本周范围内的数据。
5. 统计完成任务总数。
6. 汇总学习总时长。
7. 按学科聚合完成次数和学习时长。
8. 按日期聚合每日完成数量和每日学习时长。

展示要求：

- 无数据时显示空状态提示。
- 数据变化后实时更新。
- 学科统计需要使用学科颜色作为视觉标识。

### 6.8 月总结逻辑

月总结统计当前自然月数据。

计算步骤：

1. 获取当前本地日期。
2. 计算当月第一天和最后一天。
3. 过滤 `checkins` 中日期位于当月范围内的数据。
4. 过滤 `timerLogs` 中日期位于当月范围内的数据。
5. 统计完成任务总数。
6. 统计学习总时长。
7. 统计有打卡记录或计时记录的学习天数。
8. 按学科聚合完成次数和学习时长。
9. 计算连续打卡天数。
10. 生成月历式每日状态。

连续打卡天数计算规则：

- 从今天开始向前检查。
- 如果某天存在打卡记录或计时记录，则计入连续天数。
- 遇到第一天没有记录时停止。

## 7. UI 风格建议

### 7.1 总体风格

推荐使用简约、清爽、卡片式 UI。

设计关键词：

- 清晰。
- 安静。
- 轻量。
- 有秩序。
- 适合长期学习使用。

整体界面不应过度装饰，重点突出任务、完成状态和学习数据。

### 7.2 色彩建议

基础色建议：

- 页面背景：浅灰或近白色。
- 主文字：深灰色。
- 次级文字：中灰色。
- 主色：蓝色或绿色。
- 成功状态：绿色。
- 警示状态：橙色。
- 删除状态：红色。

学科颜色：

- 每个学科拥有独立颜色。
- 学科颜色用于标签、左侧色条、图表条形或小圆点。
- 默认提供 6 至 8 个可选颜色。

### 7.3 布局建议

桌面端：

- 页面最大宽度建议为 960px 至 1200px。
- 内容居中。
- 使用卡片承载任务、学科和统计数据。
- 顶部概览可以使用 2 至 4 列统计卡片。

移动端：

- 单列布局。
- 卡片上下排列。
- 操作按钮保持足够点击面积。
- 标签导航允许横向滚动或换行。

### 7.4 组件建议

主要组件：

- 顶部栏。
- 标签导航。
- 统计卡片。
- 学科卡片。
- 任务卡片。
- 表单输入框。
- 主按钮。
- 次按钮。
- 危险按钮。
- 空状态提示。
- 确认弹窗。
- 简易条形图。
- 月历格子。

### 7.5 交互反馈

必要反馈：

- 新增成功提示。
- 保存成功提示。
- 打卡成功状态变化。
- 删除前确认。
- 计时结束提示。
- 数据为空时显示引导。
- 表单校验错误提示。

提示方式：

- MVP 可以使用浏览器原生 `alert`、`confirm`。
- 若后续优化，可改为自定义 Toast 和弹窗。

### 7.6 可访问性建议

- 按钮文案应清晰。
- 表单控件需要对应标签。
- 完成状态不能只依赖颜色表达。
- 计时器数字需要足够大。
- 删除等危险操作需要二次确认。

## 8. 状态与空数据处理

### 8.1 无学科状态

当没有学科时：

- 今日任务页面提示用户先创建学科。
- 新增任务入口应不可用或提示需要先添加学科。
- 计时器学科选择为空时不可开始。

### 8.2 无任务状态

当没有任务时：

- 今日任务页面显示空状态。
- 提供新增任务入口。
- 周总结和月总结仍可基于计时记录展示学习时长。

### 8.3 无统计数据状态

当周总结或月总结没有数据时：

- 显示简洁提示。
- 不显示空图表。
- 提示用户完成任务打卡或使用计时器。

## 9. 数据一致性要求

### 9.1 ID 生成

所有数据对象必须拥有唯一 ID。

推荐格式：

```text
类型_时间戳_随机字符串
```

示例：

```text
subject_1717920000000_ab12cd
task_1717920000000_ef34gh
```

### 9.2 日期处理

日期展示和统计必须使用本地日期。

要求：

- 打卡日期使用 `YYYY-MM-DD`。
- 周总结按本地自然周计算。
- 月总结按本地自然月计算。
- 不应直接使用 UTC 日期作为打卡日期，否则可能出现跨时区日期错误。

### 9.3 关联数据清理

删除学科时必须清理：

- 学科本身。
- 该学科下的任务。
- 该学科相关打卡记录。
- 该学科相关计时记录。

删除任务时必须清理：

- 任务本身。
- 该任务相关打卡记录。
- 该任务相关计时记录。

## 10. MVP 验收标准

### 10.1 学科管理验收

- 可以新增学科。
- 可以编辑学科。
- 可以删除学科。
- 不允许创建空名称学科。
- 不允许创建重复名称学科。
- 删除学科时能同步处理关联数据。

### 10.2 任务管理验收

- 可以新增任务。
- 可以编辑任务。
- 可以删除任务。
- 任务必须归属于某个学科。
- 任务预计时长必须为正整数。
- 删除任务时能同步删除关联打卡记录。

### 10.3 打卡验收

- 可以对任务进行今日打卡。
- 同一任务同一天不能重复打卡。
- 可以取消今日打卡。
- 今日完成数量能正确更新。
- 刷新页面后打卡状态不丢失。

### 10.4 计时器验收

- 可以选择学科开始计时。
- 可以选择任务开始计时。
- 可以开始、暂停、继续、重置计时。
- 时间到后有响铃或明显提示。
- 计时完成后写入学习时长记录。
- 计时完成后可以选择是否同步完成任务打卡。

### 10.5 周总结验收

- 能显示本周完成任务数。
- 能显示本周学习总时长。
- 能按学科统计完成次数。
- 能按学科统计学习时长。
- 能按日期展示每日数据。

### 10.6 月总结验收

- 能显示本月完成任务数。
- 能显示本月学习总时长。
- 能显示本月学习天数。
- 能显示连续打卡天数。
- 能展示月历式打卡概览。

### 10.7 离线与持久化验收

- 页面在无网络环境下可正常打开。
- 新增、编辑、删除、打卡、计时记录可保存到 LocalStorage。
- 刷新页面后数据仍存在。
- 关闭浏览器后重新打开，数据仍存在。

## 11. 后续可扩展方向

MVP 完成后可考虑扩展：

- 数据导出与导入。
- PWA 安装与缓存。
- 自定义提醒音。
- 学习目标设置。
- 长期趋势图。
- 任务重复规则。
- 番茄钟模式。
- 每日计划模板。
- 深色模式。
- 键盘快捷键。

## 12. 开发优先级建议

推荐按以下顺序开发：

1. 项目基础结构。
2. LocalStorage 数据读写。
3. 学科管理。
4. 任务管理。
5. 今日打卡。
6. 今日概览。
7. 计时器。
8. 周总结。
9. 月总结。
10. UI 优化与响应式适配。
11. 空状态、错误提示与最终验收。

## 13. 最小可用版本边界

当以下能力全部可用时，即可认为 MVP 完成：

- 用户可以创建至少一个学科。
- 用户可以创建至少一个任务。
- 用户可以对任务完成今日打卡。
- 用户可以使用计时器并在结束时收到提醒。
- 用户可以查看本周和本月的基本统计。
- 所有数据可以在刷新和离线状态下保留。
